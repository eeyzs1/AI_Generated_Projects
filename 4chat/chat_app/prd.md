# 聊天应用产品需求文档

## 1. 产品概览
一个类似微信的简单聊天应用，支持用户注册登录、创建聊天室、实时消息传递、联系人管理等功能。

## 2. 核心功能

### 2.1 用户管理
- **注册**：用户可以注册新账户，设置用户名、显示名称、密码、邮箱，并选择或上传头像。
- **登录**：用户可以使用用户名和密码登录。
- **邮箱验证**：注册后需要验证邮箱才能激活账户。
- **密码重置**：用户可以通过邮箱重置忘记的密码。
- **个人中心**：用户可以查看和修改个人信息，包括头像、用户名、显示名称、邮箱和密码。

### 2.2 聊天室功能
- **创建聊天室**：用户可以创建新的聊天室。
- **邀请用户**：用户可以邀请其他用户加入聊天室。
- **消息发送**：用户可以在聊天室内发送文本消息。
- **实时消息**：使用WebSocket实现实时消息传递。
- **消息显示**：新消息显示在聊天记录的下端，显示发送者的显示名称和头像。

### 2.3 联系人功能
- **添加联系人**：用户可以搜索其他用户并发送联系人请求。
- **联系人请求处理**：用户可以接受或拒绝联系人请求。
- **联系人列表**：用户可以查看自己的联系人列表，按字母顺序排序。
- **联系人详情**：用户可以查看联系人的个人信息。
- **双人聊天室**：点击联系人可以进入只有两人的聊天室，如果不存在则自动创建。

### 2.4 在线状态
- **在线用户列表**：实时显示当前在线用户的列表，包括显示名称和头像。

## 3. 核心流程

### 用户注册流程
1. 用户访问注册页面
2. 填写注册信息（用户名、显示名称、密码、邮箱）
3. 选择或上传头像（支持裁剪）
4. 点击注册按钮
5. 系统发送验证邮件
6. 用户点击邮件中的验证链接
7. 账户激活成功

### 用户登录流程
1. 用户访问登录页面
2. 输入用户名和密码
3. 点击登录按钮
4. 系统验证凭证
5. 生成并返回JWT令牌
6. 前端存储令牌并跳转到聊天室页面

### 消息发送流程
1. 用户在聊天室内输入消息
2. 点击发送按钮
3. 前端通过WebSocket发送消息
4. 后端处理消息并广播给聊天室成员
5. 所有成员收到消息并更新界面

### 联系人添加流程
1. 用户访问联系人页面
2. 搜索用户
3. 发送联系人请求
4. 被请求用户收到通知
5. 被请求用户接受或拒绝请求
6. 如果接受，双方成为联系人

## 4. 用户界面设计

### 4.1 布局
- **登录/注册页面**：居中布局，简洁的表单设计。
- **主界面**：类似微信/钉钉的布局，左侧边栏显示房间列表，右侧显示聊天信息。
- **个人中心**：卡片式布局，显示个人信息和修改选项。
- **联系人页面**：列表布局，按字母顺序排序。

### 4.2 视觉设计
- **颜色方案**：主色调使用蓝色系，辅以白色和浅灰色。
- **字体**：使用无衬线字体，保证跨平台一致性。
- **图标**：使用简洁的线性图标。
- **头像**：圆形头像，大小为200x200像素。

### 4.3 响应式设计
- **桌面端**：完整的双栏布局。
- **平板端**：保持双栏布局，但宽度自适应。
- **移动端**：单栏布局，通过导航切换不同功能。

## 5. 技术要求

### 5.1 后端技术
- **框架**：FastAPI
- **数据库**：SQLite（支持MySQL）
- **认证**：JWT（双令牌机制）
- **实时通信**：WebSocket
- **部署**：Docker

### 5.2 前端技术
- **框架**：React + TypeScript
- **UI库**：Ant Design
- **路由**：React Router
- **WebSocket**：原生WebSocket API（优化了连接管理）

### 5.3 安全要求
- **密码加密**：使用bcrypt加密存储密码。
- **JWT认证**：使用JWT进行身份验证，实现双令牌机制。
- **权限验证**：确保用户只能访问自己的资源。
- **WebSocket验证**：验证WebSocket连接的合法性。

### 5.4 性能要求
- **实时性**：消息传递延迟不超过1秒。
- **响应速度**：页面加载时间不超过2秒。
- **并发支持**：支持至少100个并发用户。

## 7. 项目边界

### 7.1 功能边界
- 仅支持文本消息，不支持图片、语音等多媒体消息。
- 不支持群组视频通话。
- 不支持消息撤回和编辑功能。

### 7.2 技术边界
- 前端使用React + TypeScript，不考虑其他框架。
- 后端使用FastAPI，不考虑其他框架。
- 数据库使用SQLite，支持MySQL，但不考虑其他数据库。

### 7.3 性能边界
- 支持最多100个并发用户。
- 单个聊天室最多支持50个成员。
- 消息历史记录最多保存1000条。
